# 메모리 포렌식

## 메모리의 이해
	컴퓨터의 기본 구조 -> 폰노이만 구조 (컴퓨터 저장 방식?)
	주기억 장치와 보조기억 장치로 나뉨

	메모리 분석 이유 cpu가 연산 과정 후 나온 데이터를 얻을 수 있다.

	메모리 ( 휘발성 ) 

## 데이터 지속성
	메모리에서 사용되는 영역만 사용된다. 전부 사용 안된다(골고루 사용되지 않는다.) 따라서 복구하기가 생각보다 어렵다.
	
## 물리메모리
	물리 주소 확장(PAE)
	접근 가능한 물리 주소의 bit를 증가시키는 기술 (32bit -> 36bit) // 4GB -> 6GB
	
	### 주소 윈도윙 확장(AEW)
		>> 커널 영역(링 제로 영역) // 링 -1 영역 : 전원이 꺼진 상태
	사용자영역을 늘려줌.
	
	### 데이터 실행 방지(DEP)
	
	### 주소 공간 랜덤 배치(ASLR)
	실행되는 오브젝트들의 위치를 랜덤화
	
	### 가상 메모리
	
## 메모리의 중요성
1. 악성코드 분석( 파일리스 악성코드)
2. 사용자의 프라이버시 강화, DRM
3. 안티 포렌식 기술의 발전

## APT 공격
타겟과 다른 점?(대응 방식) 타겟 공격은 해당 타겟 공격만 치료하면된다.
APT 공격은 네트워크 전체를 대상으로 치료해야한다. (레터럴 공격?을 사용하여 계속 거점을 이동한다.)
대응하기 매우 어렵다.

## 리스트워킹
	Forward Link & Backward Link를 이용하여 프로세스를 얻는다.

## DKOM방식
	전체 Proccess 구조에서 프로세스의 오브젝트를 구조에서 제외시켜 프로세스탐지가 되지 않도록 하는 은닉 기법

### 패턴 매칭
	리스트워킹 방식의 한계를 해결하기 위한 방법
